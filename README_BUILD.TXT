# ktin_safe_full

이 버전은 첨부 원본 ktin 소스를 기준으로, 장시간 실행 시 화면 멈춤/백화와 관련될 가능성이 큰 기능만 제거하거나 비활성화한 안전판입니다.

유지한 기능:
- 파일: 빠른 연결, 주소록, 연결 끊기, 스크립트 읽기, 메모장, 종료
- 보기: 메뉴 숨기기, ANSI 테마 선택, 화면 여백 없애기, 특수기호(F4)
- 편집: 찾기, 변수, 줄임말, 단축키 설정, 타이머, 상태바, 숫자 키패드 매크로
- 편집: 지난 화면/현재 화면을 클립보드로 복사, 파일로 저장
- 옵션: 환경 설정, 단축버튼 표시/숨김, 접속 유지
- 자동 로그인: 아이디 패턴/비번 패턴 등

제거/비활성화한 기능:
- 트리거 설정 / 실시간 하이라이트 / 트리거 액션
- 채팅 캡처 패턴 설정 / 채팅 캡처창 / 채팅 캡처 시간 표시
- 자동 갈무리(계속 파일에 쌓는 방식)

렌더링 안정성 수정:
- WM_PAINT에서 셀마다 CreateSolidBrush/DeleteObject를 반복하지 않고 ExtTextOutW(ETO_OPAQUE)로 배경을 칠하도록 수정했습니다.
- 화면 렌더링 중 정규식 매칭을 하지 않도록 차단했습니다.

빌드 예시:
cl /utf-8 /std:c++17 /EHsc /O2 /DUNICODE /D_UNICODE /DNOMINMAX *.cpp Resource.rc user32.lib gdi32.lib shell32.lib comdlg32.lib comctl32.lib uxtheme.lib winmm.lib dwmapi.lib version.lib


[2026-07-07 build fix]
- /DNOMINMAX 사용 시 bare min/max가 안 보이던 문제를 constants.h에서 using std::min/max로 보정했습니다.
- settings.cpp의 제거된 채팅 캡처 줄 수 입력칸(hEditChatLines) 참조를 제거했습니다.
- build_msvc.bat가 Resource.rc를 cl에 직접 넘기지 않고 rc.exe로 Resource.res를 만든 뒤 링크하도록 수정했습니다.
- 압축에는 폰트 파일을 포함하지 않았습니다. Resource.rc의 MudDunggeunmo.ttf RCDATA 줄은 빌드 오류 방지를 위해 비활성화했습니다.

- 단축버튼이 보이지 않던 문제를 수정했습니다.

  서버 이름, 서버 주소, 서버 포트, 서버 문자셋, TinTin 스크립트만 남겼습니다.
  자동 로그인, 성공/실패 패턴, 자동 재접속은 TinTin++ 스크립트에서 처리하도록 제거했습니다.

- 옵션 일반설정에서 자동 로그인 패턴 입력 UI를 제거했습니다.

- 자동 로그인 엔진은 no-op 처리했습니다.
  화면 텍스트를 계속 감시하는 기능을 안전판에서 제거했습니다.

- 모호한 동아시아 문자 폭 기본값을 넓게 처리로 바꿨습니다.
  박스문자/도형/화살표가 깨질 때 필요한 원본 옵션을 기본 ON으로 두었습니다.

- 줄임말/타이머/변수 기능은 남겼습니다.
  이 기능들은 기본적으로 입력/예약 전송/내부 변수 관리 쪽이며, 화면을 매 줄 정규식으로 캡처하는 트리거/채팅 캡처와는 성격이 다릅니다.

- 환경설정 일반 설정에 아이디/비밀번호/성공/실패 패턴 입력을 복원했습니다.

주의:
- 이 기능은 화면을 계속 감시하지 않고 접속 직후 제한 시간 안에서만 검사합니다.
- 패턴은 정규식이 아니라 단순 포함 문자열 검사입니다.

============================================================
BUILD FIX 38 - 빠른연결/주소록 접속 전 기존 세션 종료 보강
============================================================

문제
----
- 빠른연결은 항상 `#session new ...` 형식으로 접속하지만, 기존 `new` 세션을
  GUI 상태에서 추적하지 않았습니다.
- 그래서 빠른연결을 다시 실행하거나 빠른연결 후 주소록으로 접속할 때
  기존 `new` 세션이 TinTin++ 안에 남아 있을 수 있었습니다.
- 기존 주소록 전환 코드는 주소록 세션(`hasActiveSession`)만 `#zap {세션명}`으로
  종료했기 때문에 빠른연결 세션은 종료 대상에서 빠졌습니다.

수정
----
- 실제 TinTin++ 세션명을 별도로 추적하는 필드를 추가했습니다.
  - 주소록 접속: 주소록 이름을 실제 세션명으로 기록
  - 빠른연결 접속: `new`를 실제 세션명으로 기록
- 빠른연결/주소록 접속 전에 기존 세션명이 있으면 먼저 `#zap {세션명}`을 전송합니다.
- 빠른연결에서 기존 `new` 세션을 닫은 경우에는 500ms 뒤에
  `#CONFIG {CHARSET} {...}` → `#session new ...` 순서로 실행합니다.
- 파일 메뉴의 연결 끊기도 주소록 세션뿐 아니라 빠른연결 `new` 세션까지 종료하도록 수정했습니다.
